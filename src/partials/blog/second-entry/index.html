JavaScript Async/Await: Making Promises

Date: 23.10.2022

Content:

Today's journey took me into the heart of JavaScript's asynchronous patterns, specifically the elegant async/await syntax. It's a powerful way to work with promises, making asynchronous code look and behave a bit more like synchronous code.

What I Learned:

  • Understanding Async/Await: These two keywords transform how we handle promises. I learned that marking a function with async means it always returns a promise, and await can be used to pause the code until that promise settles.

  • Error Handling: I explored using try/catch blocks within async functions for error handling, which felt more intuitive than chaining .then() and .catch().

Helpful Resources:

  • JavaScript.info on Async/Await – This resource broke down the concepts clearly and concisely.
  • Async/Await Tutorial on YouTube – This video helped solidify my understanding with practical examples.

Challenges Encountered:

  • One major challenge was refactoring existing promise-based code to async/await. It was a bit tricky to get the hang of the new syntax.

  • Dealing with asynchronous errors and understanding how to catch them effectively.

Problem Solving:

  • I dedicated time to rewrite some of my previous projects' functions using async/await, which helped me understand the nuances.

  • I practiced different error-handling scenarios to see how async/await behaves differently from traditional promise chains.

Next Steps:

  • My goal is to integrate async/await more naturally into my daily coding practices.

  • I plan to explore how async/await can be used with API calls in React components.
